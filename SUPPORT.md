# Support

Repo Context CLI is an open-source local developer tool, not a hosted service.

## Where to Ask

- Use GitHub Issues for reproducible bugs and focused feature requests.
- Use the bug report template when CLI output, generated files, or workflow behavior is wrong.
- Use the feature request template for scoped improvements that preserve the local deterministic trust boundary.

## Before Opening an Issue

- Run `repo-context pack --dry-run` when reporting `pack` behavior.
- Redact secrets, tokens, private source snippets, and private repository paths.
- Include your Node.js version and operating system.
- Include the smallest safe repository shape that reproduces the behavior.

## What This Project Does Not Provide

- Private consulting or guaranteed response times.
- Hosted processing of repository contents.
- LLM prompt debugging unrelated to generated Repo Context CLI output.
- Support for workflows that require uploading secrets or private code to an issue.

For suspected vulnerabilities, follow `SECURITY.md` instead of opening a public issue.
